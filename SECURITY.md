# Security Policy

## Supported versions
This project currently supports the latest version on the `main` branch / current release.

## Reporting a vulnerability
- **Do not** open public issues for security reports.
- Share a minimal reproduction and impact assessment.
- Include affected version/commit and environment details.

## Security principles (high-level)
- **Assume clients are compromised** (especially mobile devices).
- **Enforce authorization server-side**.
- **No secrets in the client**.
- **Lockfile installs in CI** and automated dependency monitoring.

## Vulnerability handling policy
- **Critical**: triage within 24–48 hours, prioritize fix/release.
- **High**: triage within 7 days.
- **Moderate/Low**: scheduled based on risk and exposure.
